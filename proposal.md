# 项目提案：A股每日开盘预测智能体（Daily A-Share Morning Brief Agent）

## 1. 项目概述

### 项目目标
构建一个面向证券公司员工的 AI Agent 系统，基于用户提供的证券投资顾问提示词模板，**自动生成每日 A 股开盘预测研报**。Agent 自动聚合多源金融数据（ETF 资金流向、融资融券余额、股指期货贴水、成交量、消息面等），生成符合合规要求、数据可溯源的 500 字以内专业早报。

### 应用场景
- 证券公司投资顾问每日盘前快速生成开盘预测
- 降低投顾每日手动搜集数据、撰写早报的时间（从 1-2 小时压缩到几分钟）
- 确保数据时效性、来源可追溯、用语合规

### 输入与输出
- **输入**：日期（如 2026-05-22）、用户关注的自选板块（可选）
- **输出**：500 字以内自然段落式开盘预测报告，每段末尾标注数据来源，文末汇总所有数据来源

### 最终交付物
- 可运行的 Agent 系统（Python）
- 数据采集管道（多源金融数据 API 集成）
- 报告生成流水线（LLM + 模板约束）
- 合规校验层（用语审查、数据时效性校验）
- Web Demo 或轻量前端界面

## 3. 可行性评估

### 3.1 技术可行性 ✅ 高

所需技术栈均为成熟技术：
- **语言**：Python 3.10+
- **LLM 框架**：LangChain / LangGraph（Agent 编排）
- **数据层**：AkShare（免费开源，覆盖所需全部数据类型）
- **LLM**：DeepSeek / Claude / Qwen 等支持中文的模型（API 调用）
- **前端（可选）**：Streamlit / Gradio（轻量 Demo）

### 3.2 数据可行性 ✅ 高

- 融资融券、ETF 资金流向、股指期货、指数行情等**结构化数据**均可通过 AkShare 免费获取，数据源为交易所官方
- 消息面/政策新闻**非结构化数据**可通过财联社、新浪财经等公开 API 或 RSS 获取
- 数据时效性：T+1 日盘前即可获取 T-1 日全部所需数据
- 数据质量风险：AkShare 依赖爬虫，极端情况下可能不稳定，需备选 Tushare Pro

### 3.3 模型可行性 ✅ 高

| 方案 | 难度 | 预期效果 | 适用阶段 |
|------|------|----------|----------|
| **Prompt Engineering + 单 LLM** | 低 | 基本可用，幻觉较多 | MVP |
| **多 Agent 协作 + 数据先行** | 中 | 数据准确、论点有据 | 推荐方案 |
| **SFT/RL 微调** | 高 | 风格稳定、合规性高 | 进阶优化 |

推荐采用 **"数据先行 + LLM 撰写"** 架构：所有数据计算由 Python 完成（降幻觉），LLM 仅负责将结构化数据转化为符合合规要求的自然语言。

### 3.4 算力可行性 ✅ 低门槛

- **无需 GPU 训练**：采用 API 调用 LLM，不需要本地 GPU
- **数据计算**：CPU 完全满足（轻量 Pandas 计算）
- **日均成本估算**：单次报告 LLM API 调用约 2000-4000 token，成本 < 0.1 元人民币

### 3.5 时间成本

| 周期 | 目标 | 产出 |
|------|------|------|
| **2 周 MVP** | 单 LLM + 手动粘贴数据 → 生成报告 | 可用的 Prompt 链 + 简单脚本 |
| **4 周可用版** | 自动数据采集 + 多 Agent 流水线 + 合规校验 | 完整 Pipeline + Streamlit Demo |
| **8 周完善版** | 多数据源容错 + 报告质量评估 + 历史回溯 | 稳定系统 + 评测报告 |
| **12 周产品版** | 前端界面 + 用户配置 + 定时任务 + 多格式导出 | 可部署产品 |

### 3.6 风险与备选方案

| 风险 | 影响 | 备选方案 |
|------|------|----------|
| AkShare 数据源不稳定 | 数据缺失导致报告无法生成 | 备选 Tushare Pro 或东方财富 API |
| LLM 生成内容不合规 | 合规风险 | 增加后处理规则校验 + 敏感词过滤 + 人工审核环节 |
| 消息面抓取被封 | 新闻数据缺失 | 使用财联社官方 API（付费）或 RSS 多源冗余 |
| LLM 幻觉编造数据 | 报告可信度下降 | "数据先行"架构强制所有数字来自 API，LLM 只做语言组织 |
| 合规政策变化 | 报告格式需调整 | 模板化设计，Prompt 模板和校验规则可配置 |

## 4. 深度探索路线

### 阶段 1：数据采集管道（Data Collection Pipeline）

**目标**：构建多源金融数据自动采集模块

**方法**：
- 使用 AkShare 封装统一的数据获取接口
- 实现以下数据获取器（Data Fetcher）：
  - `MarginFetcher`：融资融券余额（日频 + 近一周变化）
  - `ETFFlowFetcher`：ETF 资金流向（板块级）
  - `FuturesBasisFetcher`：股指期货升贴水（IF/IC/IH/IM 基差）
  - `IndexFetcher`：主要指数行情 + 成交量 + 均线突破检测
  - `NewsFetcher`：财联社/新浪财经昨夜消息面
  - `SentimentFetcher`：双融余额趋势、市场人气指标

**可调参数**：
- 数据回溯天数（均线计算窗口：10 日/20 日/60 日）
- 板块分类粒度（申万一级/二级行业）
- 融资余额观察周期（近一周/近一月）

**评估指标**：
- 数据获取成功率
- 数据新鲜度（时间戳校验）
- 各数据源延迟对比

**复用关系**：本阶段产出为后续所有阶段的**唯一数据入口**

---

### 阶段 2：技术指标计算引擎（Analytics Engine）

**目标**：将原始数据转化为结构化分析结论

**方法**：
- `TechnicalAnalyzer`：
  - 均线突破检测（价格突破 10 日/20 日均线）
  - 成交量放大判断（对比 20 日均量）
  - 横盘整理识别（布林带收窄 + 振幅缩小）
- `MarginAnalyzer`：
  - 融资余额趋势（近一周变化率）
  - 融资买入额占比
  - 融券余额变化
- `FuturesAnalyzer`：
  - 基差计算（期货 - 现货）/ 现货
  - 升贴水趋势（近 5 日基差变化）
- `ETFFlowAnalyzer`：
  - 板块资金净流入/流出排名
  - 连续流入/流出天数
- `NewsAnalyzer`：
  - LLM 提取关键政策/事件
  - 影响板块映射

**可调参数**：
- 均线周期选择（MA10/MA20/MA60）
- 放量阈值（1.2x / 1.5x / 2.0x 均量）
- 横盘振幅阈值

**评估指标**：
- 技术信号与事后市场走势对比（回溯检验）
- 各指标信号频率（是否过拟合）

**复用关系**：产出为 LLM 报告生成的**结构化输入上下文**

---

### 阶段 3：多 Agent 报告生成流水线（Report Generation Pipeline）

**目标**：将结构化分析数据转化为合规的 500 字早报

**方法**（参考 FinSight + 华泰助手架构）：

```
数据采集 (DataFetchers)
       ↓
技术指标计算 (Analytics Engine)
       ↓
   ┌───────────────────────────────────┐
   │  报告生成流水线 (LangGraph)         │
   │                                   │
   │  1. 大纲生成 Agent                 │
   │     - 确定今日报告重点板块/主题      │
   │     ↓                             │
   │  2. 分段撰写 Agent                 │
   │     - 技术面段落 (含数据标注)       │
   │     - 资金面段落 (ETF+双融)        │
   │     - 消息面段落                  │
   │     - 后市判断段落                │
   │     ↓                             │
   │  3. 合规校验 Agent                 │
   │     - 个股名称检测                 │
   │     - 激进用语检测                 │
   │     - 数据标注完整性校验           │
   │     - 时效性校验                   │
   │     ↓                             │
   │  4. 终稿润色 Agent                 │
   │     - 字数控制 (≤500字)           │
   │     - 自然段落分段                │
   │     - 数据来源汇总                │
   └───────────────────────────────────┘
       ↓
  最终报告 (Markdown)
```

**可调参数**：
- 每段字数分配权重
- 合规敏感词列表
- 数据来源格式模板
- LLM temperature（0.1-0.3 推荐低温度保证一致性）

**评估指标**：
- 人工评审打分（信息准确性、合规性、可读性）
- 数据标注覆盖率（应达 100%）
- 字数合规率
- 个股名称屏蔽率

**复用关系**：依赖阶段 1-2 的数据和计算产出

---

### 阶段 4：质量评估与持续优化（Evaluation & Optimization）

**目标**：建立报告质量评估体系和自动化测试

**方法**：
- 构建测试用例集（10-20 个历史交易日，覆盖上涨/下跌/震荡市况）
- 自动评估指标：
  - 数据引用准确率（对比原始数据源）
  - 合规检查通过率
  - 字数分布
- 人工评审标注（准确性 1-5 分、实用性 1-5 分）
- 错误分析分类（数据错误 / 逻辑错误 / 合规问题 / 表达问题）
- 根据评估结果优化 Prompt 模板和数据计算逻辑

**复用关系**：评估结果反馈到阶段 1-3 的参数调优

---

### 阶段 5：前端交互与产品化（可选进阶）

## 5. 实验设计

### 5.1 基线对比
- **Baseline 1**：纯 LLM（无数据先行，直接让 LLM 生成报告）
- **Baseline 2**：模板填充（人工写模板 + Python 填充数据）
- **本系统**：数据先行 + 多 Agent 生成 + 合规校验

### 5.2 消融实验
- 移除合规校验 Agent → 测量合规问题率
- 移除数据分析引擎 → 测量数据错误率
- 使用不同 LLM（DeepSeek vs Qwen vs Claude） → 测量报告质量差异

### 5.3 稳健性测试
- 不同市况表现（单边上涨/下跌/震荡/极端波动）
- 数据源缺失降级测试
- 连续多日生成一致性测试

## 8. 关键设计决策

1. **"数据先行"是降幻觉的核心**：华泰助手、FinSight、AlphaAnalyst 三个独立来源都强调了同一模式——用 Python 计算数值，LLM 只做语言组织。这是金融场景的共识方案。

2. **FinSight（ACL 2026）是最接近的学术参考**：它的 CAVM（Code Agent with Variable Memory）架构和引用强制校验机制可以直接借鉴，而且它本身就支持 A 股市场。

3. **AkShare 数据覆盖面足够**：融资融券、ETF 资金流、期货升贴水、指数行情全部覆盖，免费无限制。配合 Tushare Pro 做备选即可。

4. **合规校验需要作为独立 Agent**：考虑到证券公司合规要求的严格性，建议合规校验作为流水线中的强制关卡（gate），不通过则自动重写。

5. **500 字约束需要后处理控制**：LLM 倾向于生成更长文本，建议在润色 Agent 中硬约束字数，超出则自动压缩。

## 附录：用户原始提示词模板

```
以证券投资顾问的角度，写一篇2026年5月22日A股开盘预测。
简要概述5月21日市场走势，分析对于A股热点板块需结合ETF资金流向数据加以分析、成交量变化。关注股指期货的贴水情况。需要分析双融余额的变化，以此作为市场重要的人气指标，如果融资余额应以较长周期来分析，重点分析近一周的变化。昨夜消息面变化对市场相关板块的影响。后市如何判断？相关论点需要数据支持。数据需要标注在每自然段后。全文总字数不超过500字。
要求：所有的论点均要有数据支持。不用数字标题分段，根据文意自然分段。不要提及具体个股。要注意引用数据的时效性和真实性！不要提及具体个股名称。观点要有数据支持。
1、技术形态：参考近期主要股指突破关键均线（如10日/20日线）、成交量放大、横盘整理后启动等特征。
2、政策与情绪：结合政策利好、投资者讨论热度。
3、注意用语符合最新合规要求。既要通俗专业练达，又要避免使用过于激进的词语。
4、格式：不用标题分段，按照文意自然分段，数据标注在每自然段最后。数据标注在全文最后的（数据来源：）内
时间范围：2026年5月22日
数据标注：需要用文字标注所有数据的来源，并附在全文最后。
数据类型：市场统计/学术论文/政策法规
数据来源偏好：官方统计局/权威期刊.市况数据最好引用交易所、财联社或新浪财经；ETF数据应该采自ETF风向标、中国基金报、万得数据；市场观点数据应该采用中信证券研究、长江证券研究、银河证券研究、申万宏源研究。
部署动态时间戳：对每个回答自动标注数据溯源的更新时间且标注数据来源均需在每自然段末尾。不要标注在文中
构建时效性校验层：在输出前检测数据时间与问题要求的时间窗口匹配
市场观点参考资料：仅参考ETF风向标、中国基金报、财联社、FTAA、猫笔刀、渐近投研、寻暇记、蟹壳、小韭菜量投、洪灏的宏观策略、长江另类策略、中信证券研究、每日经济新闻、证券日报等微信财经公众号当日的信息。
```
